GTM 总览

一句话 GTM：Wuji 的 GTM 是“底层电机能力 -> 灵巧手硬件平台 -> 手套/数据/重定向工作流 -> B2B RFQ -> 客户交付和全球业务”的硬科技商业化路径。

舞肌的 GTM 不应该套普通消费品牌模板。它更像一条 B2B 硬科技路径：先用高扭矩密度电机能力建立底层执行器优势，再把 Wuji Hand、Wuji Hand 2、Wuji Glove、SDK/Studio、重定向、仿真和客户交付包装成开发者与人形机器人团队可评估的平台。官网负责建立产品入口，论文负责证明真实任务能力，招聘负责暴露商业化组织建设，RFQ 和交付支持负责把技术转成订单。当前最大的缺口是价格、客户、收入和 ROI，因此 GTM 报告必须把技术可信和商业回报分开。

渠道路径

官网产品页建立 Wuji Hand / Hand 2 / Glove 认知→论文和项目页证明真实任务、重定向和安全控制能力→Docs / SDK / Studio 降低开发者评估门槛→Cooperation / Contact 进入 RFQ 和技术销售→交付、质量、制造和全球业务岗位支持客户部署→用客户案例、报价、交期和售后数据验证商业化

关键学习

  • 原始洞察：人形机器人需要能完成真实抓取、旋转、笔旋转等接触丰富任务的末端执行器；只靠普通夹爪或低自由度手无法承载复杂示教和操作数据。
  • 旧方案缺口：只会开合的手无法满足 pen spinning、cube reorientation、遥操作和数据采集需求。
  • 定价/商业模式：公开价格未披露，按 RFQ 处理，并拆分硬件、accessory、software、service 和 warranty。
  • 增长触发：官网产品入口 -> 论文验证
  • 客户收益：成本降低尚未被公开价格证明；必须和 Shadow、Allegro、LinkerBot、自研方案做总拥有成本对比。
  • 可学习：硬科技 GTM 的第一步是用真实任务验证建立信任，而不是先做大规模营销。

关键学习

  • 原始洞察：灵巧手需要人类示教和遥操作数据，但人手动作映射到机器人手会产生接触关系丢失、自碰撞、穿透和不可执行姿态。
  • 旧方案缺口：客户要的是可交付的采集、重定向、仿真、安全和真实硬件执行流程。
  • 定价/商业模式：公开价格未披露，按 RFQ 处理；software 和 service 边界要和硬件分开。
  • 增长触发：手套产品页 -> 手部数据采集
  • 客户收益：成本降低需要报价和项目工时证明；开源软件会压低纯软件收费预期。
  • 可学习：灵巧手公司的第二曲线可能不是另一只手，而是数据采集和部署工作流。

综合学习总结

最重要启发

  • 舞肌的学习价值在于从底层电机能力切入，再扩展到灵巧手平台，而不是单纯讲机器人概念。
  • Wuji Hand 的商业化要看完整报价栈：本体、配件、软件、服务、保修和备件。
  • Wuji Glove 和数采平台岗位说明数据入口可能是第二增长层。
  • 找不到的价格、收入、亏损、客户和 ROI 必须留作缺口，不能写成结论。

仍需确认的问题

  • Wuji Hand / Hand 2 / Glove 的正式报价、交期、保修和支持费是多少？
  • SDK、Studio、仿真、重定向和远程诊断是否单独收费？
  • 是否已有付费客户、出货量、收入、毛利和复购证据？
  • 中国移动入股的金额、股权比例和战略协同是什么？
  • 数采平台岗位是否会发展为独立数据产品或客户数据服务？
