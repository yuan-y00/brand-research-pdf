# 创业案例研究提示词

你要生成的是给创业者学习的案例，不是泛泛的品牌介绍。

读者是创业者。目标是看懂：这个产品为什么被开发、解决了什么痛点、开发和上市花了什么代价、创造了什么回报、ROI 能不能被计算，以及从官网、招聘、渠道、专利、融资和收入线索里能推断公司下一步要做什么。

除品牌名、产品名、英文引用标题和必要的专业缩写外，正文必须用中文。找不到的事实必须写成“未找到公开证据 / 不可计算 / 需要继续验证”，不能用英文占位。

## Required Output

返回符合 `data/cases/_schema.json` 的合法 JSON。

## Research Rules

1. 不要编造创始人自有资金、研发成本、BOM、CAC、毛利率、退款损失、质保准备金、实时估值或 ROI。
2. 如果 ROI 不能计算，把 `roi_assessment.calculable` 设为 `false`，并列出缺失输入。
3. 招聘信息只能作为方向推断。必须记录它是已捕捉岗位、招聘索引页，还是尚未捕捉到稳定 JD 的观察项。
4. 官网信号必须具体：新品页、价格变化、技术关键词、服务条款、经销商页、支持页、投资者页、地区或渠道扩张。
5. 每个核心产品案例都要解释：
   - 为什么开发它
   - 原始用户痛点
   - 为什么老玩家没做好或行动慢
   - 开发难点
   - 已知开发成本信号
   - 商业化路径
   - 产品、配件、服务、软件订阅和竞品价格
   - 回报证据
   - ROI 判断
   - 创业者应该学什么

## Report Type

选择一种：

- `product_company_case`：产品驱动公司，例如 Mammotion、Shokz、Speediance。
- `platform_company_case`：平台型公司，例如 Apple、WeChat、Tesla / SpaceX。
- `person_method_case`：人物 / 产品方法论案例，例如 Jobs、张小龙。
- `gtm_comparison_case`：同一品类的横向对照案例。

## Evidence Standard

优先使用一手来源：

- 官方产品页
- 官方支持页 / 价格页
- SEC / 港交所 / 年报
- Kickstarter 活动页
- 公司招聘页或带岗位标题的招聘平台页面
- 专利或监管文件

只有一手来源不可得时，才使用可信媒体；使用二手来源时，可信度标为 `medium` 或 `low`。
